l = []
knns = []
data = heart.drop("target", axis=1)
target = heart["target"]
for n in range(1,51):
    l.append([])
    knns.append([])
    for i in range(0,100):
        knn = KNeighborsClassifier(n_neighbors=n)
        X_train, X_test, y_train, y_test = train_test_split(data,
                                                    target,
                                                    test_size=0.3)
        knn.fit(X_train, y_train)
        knns[-1].append([knn, y_test, knn.predict(X_test)])
        l[-1].append(metrics.accuracy_score(y_test, knn.predict(X_test)))

l = []
logregs = []
data = heart.drop("target", axis=1)
target = heart["target"]
for i in range(0,100):
    logreg = LogisticRegression(random_state=0,
                        solver='lbfgs',
                        multi_class='multinomial',
                        max_iter=5000)
    X_train, X_test, y_train, y_test = train_test_split(data,
                                                target,
                                                test_size=0.3)
    logreg.fit(X_train, y_train)
    logregs.append([logreg, y_test, logreg.predict(X_test)])
    l.append(metrics.accuracy_score(y_test, logreg.predict(X_test)))

Conclusion¶

On constate que les modles qui demandent le plus de ressources sont aussi plus prcis dans notre cas. Mais aussi que diffrents classifieurs sont adapts diffrents types de jeux de donnes. Avec plus de donnes il faudrait donc trouver un compromis entre vitesse d'excution et performance.
Dans notre cas des classifieurs plus simples et linaires ont donns de meilleurs rsultats.
